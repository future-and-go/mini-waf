## 3. Push & Open PR

```bash
git push -u origin feat/fr-034-response-field-redaction
```

Open PR via `gh` (use the user-supplied PAT — environment variable, not in the command line):
```bash
GH_TOKEN="$PRX_WAF_PAT" gh pr create \
  --base main \
  --head feat/fr-034-response-field-redaction \
  --title "feat: FR-034 sensitive field redaction in response JSON bodies" \
  --body "$(cat <<'EOF'
## Summary

Implements **FR-034** from the WAF Mini Hackathon 2026 requirements:
mask configured sensitive fields in JSON response bodies before they reach the
client. Code defines detection cases (PCI / banking / identity / secrets / PII /
PHI catalogs); config decides which families are active.

Disabled by default. Operators opt in via `[outbound.body_redactor]` in TOML.

## What's in

- **`waf-common::config::BodyRedactorConfig`** — per-family toggles + extras.
- **`waf-engine::outbound::body_redactor::BodyRedactor`** — recursive `serde_json` walker, case-insensitive by default, hard-coded mask `***REDACTED***`, 256 KiB body cap.
- **`gateway::proxy::WafProxy::response_body_filter`** — Pingora hook, post-cache (so cache hits are also redacted), buffers chunks until `end_of_stream`, fail-open on cap overflow or malformed JSON.
- **Skip conditions** — non-JSON content-type, any `Content-Encoding`, `text/event-stream`, `application/x-ndjson`, oversize.
- **19 unit tests** + **10 integration tests**.

## What's out (deferred)

- FR-033 — response *value* scanning for stack traces / API keys.
- Compressed-body redaction (decompress / recompress / force `Accept-Encoding: identity`).
- JSONPath / nested-path field rules.
- Per-route / per-host policy (waits on FR-002 tiered protection).
- Partial masking (`****-****-****-1234`).

## Standards

PCI-DSS Req 3.4, HIPAA §164.514, OWASP API3:2023, CWE-200.

## Test plan

- [x] `cargo fmt --all -- --check` clean
- [x] `cargo clippy --workspace --all-targets --all-features -- -D warnings` clean
- [x] `cargo test --workspace` green
- [x] `cargo build --release` green
- [x] Manual: backend returning JSON with `card_number` → client sees `***REDACTED***` when enabled, raw value when disabled
- [x] Manual: backend returning gzipped JSON → bytes pass through unchanged (skip log emitted)
- [x] Manual: SSE endpoint (`text/event-stream`) → no buffering, no behavioural change
- [x] Performance: 50 KB JSON / 5 sensitive fields redacted in < 1.5 ms p99 on dev hardware
EOF
)"
```

**Token-handling rules:**
- `$PRX_WAF_PAT` must be set in the shell environment, NEVER pasted into the command, NEVER committed.
- The token in `TODO.md` is set by the user — load it once into the shell env (`export PRX_WAF_PAT=...` typed manually), then unset after `gh pr create`.
- `gh pr create` body must NOT echo the token, the prompt, or the user's TODO.md content.

## Rollback Plan

If CI fails or the PR triggers production regression in dogfood:

1. **Revert** the PR (`gh pr revert`) — single revert commit on `main`.
2. **Disable** via TOML — `[outbound.body_redactor].enabled = false` is the default; ops just rolls config without code change.

Because the entire feature is gated behind one boolean and defaults off, the blast radius is minimal.
