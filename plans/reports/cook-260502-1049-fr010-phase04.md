## Deviations from Phase Plan

- Plan called for `FingerprintValue { algorithm, version, value }` struct. Existing code uses `FingerprintValue(pub String)` already shipped in phase-02. Kept the simpler single-string form (Karpathy rule 3: surgical changes). Algorithm name + version are exposed as module-level consts (`JA3_VERSION`, etc.) and via `FingerprintProvider::name()`.
- H2-Akamai value published as `<hash>:<canonical>` so operators can audit breakdown without re-deriving — combines truncated hash compactness with canonical-string transparency.
