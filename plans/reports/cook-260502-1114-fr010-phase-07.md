# Phase-07 — RiskAggregator + DeviceFpDetector::process Wiring

**Date:** 2026-05-02 | **Branch:** feat/fr-010 | **Plan:** 260501-2005-fr010-device-fingerprinting/phase-07

## Summary

Shipped end-to-end device-fingerprint pipeline + FR-025 plug-in seam.

## Changes

### `crates/waf-engine/src/device_fp/aggregator.rs`
- FR-025 plug-in contract rustdoc (semantics, skeleton impl).
- `NoopAggregator::submit` now emits `tracing::debug!` w/ key + signal count.
- New `LoggingAggregator` — bounded `VecDeque` ring buffer (`Arc<Mutex<…>>`), cheap clone for assertion sites.
- `AggregatorSubmission { key, signals }` snapshot type.

### `crates/waf-engine/src/device_fp/mod.rs`
- `DeviceFpDetector` extended w/ `fingerprints: FingerprintRegistry`, `store: Option<Arc<dyn IdentityStore>>`, `aggregator: Arc<dyn RiskAggregator>` (default `NoopAggregator`).
- Builders `with_store`, `with_aggregator`.
- New async `process(peer_ip, ua, conn) -> DeviceIdentity`: assemble fp → observe (skip on empty key or missing store) → dispatch providers → fire-and-forget aggregator submit. Fail-open: store errors logged at `warn`, request proceeds.
- Re-exports: `AggregatorSubmission`, `LoggingAggregator`.

### `crates/gateway/src/proxy.rs` + `context.rs`
- `WafProxy.device_fp_detector: Option<Arc<DeviceFpDetector>>` + `with_device_fp_detector` builder.
- `request_filter` invokes `process()` after relay detector. Uses validated `client_identity.real_ip` when present, falls back to TCP peer. UA from `User-Agent` header. ConnCtx is fresh `ConnCtx::new()` until L4 capture wiring lands (phase-03-sub) — empty `FpKey` → no observe; UA-only providers still fire.
- `GatewayCtx.device_identity: Option<DeviceIdentity>`.

### `crates/waf-engine/tests/device_fp_pipeline_e2e.rs`
- `process_emits_h2_anomaly_and_ua_blocklist_into_aggregator`: synthetic ConnCtx w/ zero-increment `WINDOW_UPDATE` + UA `curl-impersonate/1.0` against blocklist `(?i)curl-impersonate` → asserts both `H2Anomaly::ZeroWindowUpdate` + `UaBlocklisted` reach `LoggingAggregator`.
- `process_with_default_aggregator_does_not_panic` smoke test for boot path.

### `docs/system-architecture.md`
- New "Gateway → DeviceFpDetector (FR-010)" section: pipeline diagram + FR-025 plug-in contract + skeleton impl.
